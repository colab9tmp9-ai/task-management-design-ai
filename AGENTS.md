# AGENTS.md

このリポジトリで作業するAIエージェント向けのルール。

---

## 基本原則

このリポジトリは、個人用スケジュール・タスク管理システムの
設計を管理するための公開リポジトリである。

AIエージェントは以下の原則に従うこと。

1. ユーザーの明示的な依頼がある場合にのみ作業する
2. 変更が必要と判断した場合でも、まず提案を提示し、ユーザーに許可を求める
3. ユーザーの明示的な許可があるまで実行しない
4. 個人情報・固有名詞を含むファイルをGitHubに追加しない

---

## 提案と実行の分離

AIエージェントは、変更が必要だと判断した場合でも
ただちに実行してはならない。

まず以下の形式で変更案を提示し、
ユーザーの明示的な許可を得てから実行すること。

​```text
## 変更提案

対象ファイル：
変更前の内容：
変更後の内容：
変更理由：
影響範囲：
個人情報・セキュリティ上の懸念：
実行してよいですか？
​```

---

## 禁止事項

ユーザーの明示的な許可なしに以下を行ってはならない。

- ファイルの追加・削除・移動・リネーム
- 既存ファイルの大幅な書き換え
- GitHub Actions・workflowの追加・編集
- 外部サービス連携の追加
- APIキー・トークン・パスワード・認証情報の作成・保存・参照
- 個人情報・固有名詞のGitHubへの追加
- 実際の予定・タスクの追加
- 新しい依存関係・パッケージ・外部API利用の追加
- 自動実行されるスクリプトの追加
- リポジトリ設定の変更
- ライセンスの追加・変更
- Issue・Pull Request・workflow・releaseの作成

該当する作業が必要な場合は、以下の形式で確認を求めること。

​```text
## 操作提案

対象ファイル：
操作前の内容：
操作後の内容：
変更理由：
影響範囲：
個人情報・セキュリティ上の懸念：
実行してよいですか？
​```

---

## 許可される作業

以下を行ってよい。

- Markdown文書の誤字修正の提案
- 設計文書の改善案の作成（個人情報を含まないもの）
- ファイル構成のレビュー
- 抽象的な設計案の作成
- Pull Request案の説明
- 変更前後の差分説明

---

## 個人情報の扱い

このリポジトリはpublicのため、以下の情報を含めてはならない。

- 実在する企業名・学校名・研究室名
- メールアドレス・アカウントID
- 個人名
- 面接日時・実際の締め切り
- Slack・メールの本文
- APIキー・トークン・パスワード
- 非公開URL

### 仮称ルール

GitHubのファイルに固有名詞を書く場合は、実名ではなく仮称を使うこと。
仮称と実名の対応表は、ユーザーがGitリポジトリ外の非公開場所で管理する。
AIエージェントは、ユーザーが明示的に必要な範囲だけ提示した場合を除き、
仮称対応表を読もうとしてはならない。
publicリポジトリには、仮称と実名の対応表・対応表の内容・
保存場所・ローカルパスを書いてはならない。

| カテゴリ | 仮称の形式 |
|---------|-----------|
| 就活先 | 就活先A・就活先B… |
| インターン先 | インターン先A・インターン先B… |
| 大学の講義 | 授業A・授業B… |
| メールアカウント | メールアカウントA・メールアカウントB… |

AIエージェントは、実名や対応関係を推測して書いてはならない。

対応表にない固有名詞が必要な場合は、以下の形式でユーザーに確認を求めること。
ユーザーが許可した場合のみ、対応表に追記する。

​```text
## 仮称追加提案

追加する仮称：（例：就活先E）
対応する実名：（例：○○株式会社）
カテゴリ：（例：就活先）
仮称対応表に追記してよいですか？
​```

## 作業前チェックリスト

変更を行う前に以下を確認すること。

1. 変更対象ファイルを特定したか
2. 変更内容の要約を用意したか
3. 個人情報・固有名詞を含まないか
4. セキュリティ上危険な変更を含まないか
5. Pull Requestを作成する場合、ユーザーの許可を得たか
