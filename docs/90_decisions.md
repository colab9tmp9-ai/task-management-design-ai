# 決定事項

このファイルには、採用済みの設計判断だけを書く。

AIの提案や議論の途中経過は `99_ai_discussion_log.md` に書く。迷っていることは `91_open_questions.md` に書く。

## 2026-04-26

### GitHub運用

- 設計ログ置き場として、publicリポジトリを使う。
- リポジトリ名は `task-management-design-ai` とする。
- GitHubには公開してよい設計情報だけを書く。
- 実際の予定、タスク、企業名、学校名、面接日時、メール本文、Slack本文などはGitHubに書かない。
- ChatGPTとClaudeには、このリポジトリのファイルを参照させる。

### 情報の置き場所

- 設計ドキュメントはGitHubに置く。
- 実際のイベントはGoogleカレンダーに置く候補とする。
- 実際のタスクはNotionに置く候補とする。
- 個人情報や実データは、GitHubではない非公開の場所に置く。

### AIの役割

- ChatGPTとClaudeは、設計の相談、比較、レビューに使う。
- 両AIの回答は、全文ではなく要約して記録する。
- 採用する内容は人間が判断する。
- 採用済みの内容だけをこのファイルに移す。
