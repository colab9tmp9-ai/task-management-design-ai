# ツール選定

## 目的

このファイルでは、スケジュール・タスク管理システムで使用するツールの役割を整理する。

## 候補ツール

現時点の候補は以下である。

- Googleカレンダー
- Notion
- GitHub
- Pythonスクリプト
- ChatGPT
- Claude

## Googleカレンダー

### 役割

日時が決まっているイベントの正本として使う。

### 管理する情報

- 授業
- 面接
- ミーティング
- 勤務
- 研究室予定
- 移動や参加が必要な予定

### 利点

- 通知が強い
- スマホで確認しやすい
- 日時管理に向いている
- 繰り返し予定を扱いやすい

### 注意点

- タスクの詳細管理には向かない
- メモを詰め込みすぎると見づらい
- 完了状態の管理には弱い

## Notion

### 役割

タスク、案件、メモの管理候補として使う。

### 管理する情報

- タスク一覧
- 締め切り
- ステータス
- 優先度
- 関連案件
- 提出方法
- メモ
- 確認待ちタスク

### 利点

- データベース形式で整理しやすい
- スマホから編集できる
- Notion APIでPython連携できる
- タスク、案件、メモをまとめやすい

### 注意点

- 作り込みすぎると運用が重くなる
- 通知はGoogleカレンダーほど強くない
- データベース設計を複雑にしすぎない必要がある

## GitHub

### 役割

公開してよい設計ドキュメントの置き場として使う。

### 管理する情報

- 要件整理
- 分類ルール
- ツール選定
- 決定事項
- 未決定事項
- AI相談ログ
- 将来のPythonスクリプト

### 利点

- ChatGPTとClaudeに同じ文書を共有しやすい
- Markdownで管理できる
- Gitの練習になる
- 将来スクリプト管理にも使える
- 変更履歴が残る

### 注意点

- publicリポジトリには個人情報を書かない
- 実際の予定やタスクは置かない
- APIキーやトークンを絶対に置かない

## Pythonスクリプト

### 役割

日常的な自動化に使う。

### 候補となる処理

- 朝のサマリー作成
- Googleカレンダーから今日の予定を取得
- Notionから期限が近いタスクを取得
- メールやチャットから候補タスクを抽出
- Slackまたはメールへ通知

### 方針

最初は完全自動登録を目指さない。

まずは、候補を抽出し、人間が確認する形にする。

## ChatGPT / Claude

### 役割

毎日の自動化ではなく、設計レビューや相談に使う。

### 使う場面

- システム設計の相談
- Notion DB設計の比較
- 優先順位の相談
- 返信文の作成
- AI相談ログの要約
- 設計文書のレビュー

### 方針

- 日常運用の中心にはしない
- 週1回程度の整理に使う
- 回答は要約してGitHubに残す
- 採用するかどうかは人間が決める

## 現時点の仮決定

- イベントはGoogleカレンダーで管理する
- タスクはNotionで管理する候補とする
- 設計ログはGitHub public repositoryで管理する
- 実データはGitHubに置かない
- Pythonスクリプトで日常の自動化を行う
- ChatGPTとClaudeは設計・相談・レビューに使う
