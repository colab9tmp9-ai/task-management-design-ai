# task-management-design-ai

このリポジトリは、個人用のスケジュール・タスク管理システムを設計するための公開ドキュメント置き場です。

主な目的は、ChatGPT と Claude が同じ設計資料を参照できるようにし、両者の提案を比較・統合しながら、段階的に設計を固めることです。

## 対象範囲

このリポジトリでは、以下を扱います。

- スケジュール・タスク管理システムの要件整理
- イベント、タスク、案件の分類ルール
- Googleカレンダー、Notion、GitHub、Pythonスクリプト等の役割分担
- 自動化方針
- AIとの相談ログ
- 採用した設計上の決定事項
- 未決定の論点

## 公開リポジトリとしての注意

このリポジトリは public で運用するため、個人情報や具体的な予定は書きません。

書かない情報：

- 実在する企業名
- 面接日時
- メールアドレス
- 学校名
- 研究室名
- Slackやメールの本文
- 担当者名、先生、学生などの実名
- APIキー、トークン、パスワード
- 実際の予定や締め切りの詳細

代わりに、抽象化した名前を使います。

例：

- 企業A
- 企業B
- メールアカウントA
- メールアカウントB
- 大学アカウント
- 研究室チャット
- 授業A
- インターンA

## AIへの共有方法

ChatGPT や Claude に相談するときは、このリポジトリの該当ファイルの GitHub URL または raw URL を渡します。

基本的には、以下のファイルを読ませます。

- `docs/01_public_requirements.md`
- `docs/02_event_task_rules.md`
- `docs/90_decisions.md`
- `docs/91_open_questions.md`
- `docs/99_ai_discussion_log.md`

AIから得た回答は、そのまま全文を貼るのではなく、要約して `docs/99_ai_discussion_log.md` に追記します。

採用した内容だけを `docs/90_decisions.md` に移します。

## 基本方針

- GitHubには設計だけを書く
- 実際の予定やタスクは、Notion、Googleカレンダー、または非公開の場所に置く
- 日常の自動化は、できるだけPythonスクリプトで行う
- ChatGPTやClaudeは、設計レビュー、優先順位相談、文章作成補助に使う
- 最初から完全な仕組みを作らず、段階的に設計する
